# Roadmap / 路线图

## Summary / 摘要

Phase 1 runs on a 6-week cadence.  
第一期按 6 周推进。

Primary goals:

- ship one submission-ready SHIT-style paper
- publish an 8-post Xiaohongshu series
- release reusable templates and scripts
- reach 30+ GitHub stars as the floor target

## Milestones / 里程碑

### v0.1 Launch Kit

- README, manifesto, roadmap, contributing guide
- phase scaffolding
- first two Xiaohongshu-ready share packs
- issue templates and helper scripts

### v0.2 Style Study

- SHIT tone and structure study
- topic matrix
- default lead topic selected

### v0.3 Topic Vote

- public vote issue and share post
- locked lead topic
- method protocol outline

### v0.4 Draft Lab

- abstract v1
- outline v1
- figure concepts
- draft excerpts and weekly recap

### v0.5 Submission-Ready

- final abstract, intro, methods, results, figures, references
- release notes
- submission-ready package in `paper/final/`

## Weekly Execution / 周执行节奏

### Week 1

- launch repo skeleton
- publish origin story
- publish SHIT style study
- open topic vote

### Week 2

- lock topic
- publish topic matrix and rationale
- produce first method protocol

### Week 3

- build fake dataset design
- sketch figures
- write abstract v1

### Week 4

- draft intro + methods
- publish a process recap
- gather audience reactions for revisions

### Week 5

- draft results + discussion
- package figures and references
- prepare submission-ready files

### Week 6

- final polish
- publish final recap
- cut `submission-ready` release

## Success Metrics / 验收指标

By the end of week 6:

- 8 Xiaohongshu posts
- 4 GitHub milestone releases
- 1 full paper package
- 30+ stars baseline
- at least 1 audience decision reflected in the paper or roadmap

## Current Status / 当前状态

- `Phase 00`: done
- `Phase 01`: done
- `Phase 02`: in progress

## Risks / 主要风险

- official SHIT site remains suspended
- audience growth may outpace writing speed
- the repo may drift into pure storytelling if reusable assets are not maintained

## Mitigations / 对策

- define `submission-ready` as a valid end state even without live submission
- make every shareable post map to a repo artifact
- keep reusable templates and scripts visible in the top-level docs
