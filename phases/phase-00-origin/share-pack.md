## Caption / 正文文案

我最近决定开一个新项目，名字先叫 `VIBE SHITTING`。  
不是玩梗式记录，也不是随便写点“厕刊笑话”。

我想做的是：真的用 vibe coding 的方式，公开造一篇 SHIT 风格论文，并且把每一步都留档到 GitHub 上。

为什么要单独做一个新项目？

因为它和我之前的 `codex` 不是一个东西。  
旧项目更像实验和方法论；这个项目更像公开创作、公开发疯、公开包装，但仍然要讲方法。

接下来我会把：

- 选题怎么定
- 方法怎么装得像真的
- 图表怎么做得又假又严谨
- 最后怎么包装成投稿包

都完整存下来。

如果你也想看“怎么把荒谬写得像论文”，可以一起看我把这个项目做起来。
